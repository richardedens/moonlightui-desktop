How to install?

You can install MoonlightUI via GIT with the following command.


# git clone https://github.com/richardedens/moonlightui.git

How to setup?

MoonlightUI is MoonlightUI jsCore on top off jQuery with a set of libraries packed into one JavaScript and one CSS file. Both files are ready for you to use in your production environment. Take a look at the code to your right to start implementing MoonlightUI. At the bottom of this page you will find a list of all the libraries that are included.

MoonlightUI will not conflict with any other jQuery implementation. Because we use the "no conflict" option of jQuery. It is however wise to eighter at the top of all your scripts or at the bottom of all your script implement moonlightui.min.js. 


<!DOCTYPE html>
<html>
    <head>
        <script type="text/javascript" src="dist/js/moonlightui.min.js"></script>
        <link href="dist/css/moonlightui.min.css" media="all" rel="stylesheet"></link>
    </head>
    <body>
        ... markup for moonlighui ...
        <script type="text/javascript" src="app.js"></script>
    </body>
</html>

How to use?

MoonlightUI components are a set of HTML elements that have MoonlightUI CSS class names on them.

MoonlightUI is borrowing allot of concepts of AngularJS 1.x. One of them is that the markup also contains special data attributes that create interactivity between the MoonlightUI components.

Notice how in the following example we have added "data-ml-tab" as an extra attribute to the LI. This will create an onclick interaction on the tab to show the content of a div with the ID "tab1".

<li class="moonlightui-tab active" data-ml-tab="tab1"></li>

To activate the MoonlightUI functionality in your interface, use the following code on the right in your own startup script file.

MoonlightUI is still in the development fase. So CSS classnames or HTML markup might change rappidly.


/* MOONLIGHT UI */
moonlightui().onready(function() {

    /* MOONLIGHT UI - Start using moonlightui. */
    moonlightui().energize('.moonlightui');

});

External libraries

Aldo MoonlightUI in its core is jQuery. Some external libraries need to be loaded completely before extra scripts that you create can be executed. To know when MoonlightUI is ready you must use: 

moonlightui().onready(function(){});

All external libraries are already weaven on top of MoonlightUI. In MoonlightUI you have jQuery UI / jsTree / jsPlumb / async and PrismJS.


/* MOONLIGHT UI - use the onready to have also jsPlumb be loaded and ready */
moonlightui().onready(function(){
    console.log('Show that jsPlumb is loaded');
});

/* MOONLIGHT UI - AsyncJS Example */
moonlightui().async.waterfall([
    function(cb) {
        console.log('This will be printed in the console.log.');
        cb();
    }
}, function(){
   console.log('Done with the async waterfall method.');
});

Modules, Controllers and Actions

MoonlightUI allows any HTML element to have a "data-ml-action" or "data-ml-click" attribute that can execute JavaScript when clicking on the element.

But before it can actually execute JavaScript code you need to add a "data-ml-module" and "data-ml-controller" to eighter the same or a parent HTML element to set the registered module and controller to use when executing the JavaScript code.

Once you have done that you will also need to create the module and controller in JavaScript. This is done almost the same as in AngularJS 1.x. The good thing is that MoonlightUI is NOT AngularJS so you can combine both AngularJS with your MoonlightUI JavaScript code together in one project.

<div data-ml-module="core" data-ml-controller="mainController" data-ml-action="demoShowDialog"></div>

/* Action on the global scope  */
moonlightui().module('core').controller('mainController', function(){

    var mainController = {

        demoShowDialog: function(element) {
            moonlightui('#demoDialog').css('top', (moonlightui(element).offset().top) + 'px');
            moonlightui('#demoDialog').css('left', (moonlightui(element).offset().left) + 'px');
            moonlightui('#demoDialog').css('margin-left', '0px');
            moonlightui('#demoDialog').removeClass('hidden');
        }

    };

    return mainController;

});

Models

MoonlightUI has the ability to also use models and two-way databinding.

Models are registered to the module, not the controller. This means that multiple controllers can use the same model to store data and process it to the backend.

This implementation of MVC is very light weight. But can do just the trick to attach multiple MoonlightUI components together. On the right you will see a small example in inaction.

The following example will show two-way databinding in action.

<div data-ml-module="demo" data-ml-controller="demoController">
    <div>
        <input type="text" data-ml-model="demoModel.greeting" value="">
        <span data-ml-model="demoModel.greeting"></span>
    </div>
</div>

/* Action on the global scope  */
moonlightui().module('demo').model('demoModel', function(){

    /*
     * The model is extended with functions so you can set and get properties and receive UI changes.
     */
    var model = {
        greeting : '';
    }

    return model;
});

moonlightui().module('demo').controller('demoController', function(){

    /*
     * Using a model can be achieved by using the method getModel.
     * Notice that you can also easily grab a model from a different module?
     * The first parameter is the module.
     * So you can use this model in every controller of every module if you wish.
     */

    var demoController = {

        init: function() {

            var demoModel = moonlightui().getModel('demo','demoModel');

             demoModel.receive(function(name){
                 console.log('Got new value from "' + name + '": ' + demoModel.get(name));
             });

        }

    };
    demoController.init();

    return demoController;

});

Views

MoonlightUI also enables you to use views.

Views also have two-way databinding, actions and all the that moonlightui provides.

A view begins with a state and its rendering is done at this stage serverside. MoonlightUI does not have a template engine yet but has the ability for you to manipulate the outcome of the HTML inside the render function.

The following example will show the rendering of views.

one
two 
<div id="theView"></div>

/* Action on the global scope  */
moonlightui().module('demo').view('mainView', function(){

    var mainView = {
        container: 'theView',
        templateURL: '...url to template...'
    };

    return mainView;

});

/* Using a view inside your controller. */
moonlightui().module('demo').controller('demoController', function(){

    /*
     * When you registering a view it's automaticly added to the HTML.
     * And all the actions, tab switchs, tabs, scrollbars etc are initiated.
     * Using views helps with registering and unregistering moonlightui events on a view.
     */

    var demoController = {

        init: function() {

             var viewOne = moonlightui().getView('demo','viewOne');

             viewOne.render(function(template, container) {

                // You will get in the callback the template and container id.

             });

        }

    };
    demoController.init();

    return demoController;

});

Integrating with laravel

MoonlightUI has a helper function to swiftly connect to laravel.

Especially when using POST's, PUT's and DELETE's you will need a csrfToken.

The following code on the right can be added to the page to prepare MoonlightUI to send GET / POST / PUT / DELETE calls to a Laravel API.


<!DOCTYPE html>
<html>
    <head>
        <script type="text/javascript" src="dist/js/moonlightui.min.js"></script>
        <link href="dist/css/moonlightui.min.css" media="all" rel="stylesheet"></link>
    </head>
    <body>
        ... markup for moonlighui ...
        <script type="text/javascript">
            moonlightui().module('moonlightui').model('settings', function() {
                return {
                    'csrfToken': '{{ csrf_token() }}';
                };
            });
        </script>
        <script type="text/javascript" src="app.js"></script>
    </body>
</html>
                

/* Using a view inside your controller. */
moonlightui().module('demo').controller('demoController', function(){

    /*
     * To send something to the server simple use the following code:
     */

    var demoController = {

        sendToServer: function() {

             moonlightui().laravel.http.post(

                    // The API url
                    'http://....',

                    // The data to send
                    {
                       ... data to send in json format ...
                    },

                    // The done function
                    function() {
                        ... do something ...
                    },

                    // The fails function
                    function() {
                        ... do something on fail ...
                    }

             );

        }

    };

    return demoController;

});
